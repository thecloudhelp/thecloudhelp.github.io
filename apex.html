What is Apex?

Apex is a proprietary programming language used by the Salesforce platform to develop custom functionality, such as triggers, classes, and controllers. It is an object-oriented language and is similar to Java in syntax and structure. Apex is used to create custom business logic, automate complex processes, and integrate with external systems. It also has features like exception handling, collections, and database integration. The code is stored on Salesforce servers and is executed in the cloud. The main advantage of using Apex is that it allows developers to extend the functionality of Salesforce beyond what is available out of the box.

It is important to note that Apex is a strongly-typed language, which means that the data type of a variable must be declared explicitly and cannot be changed during runtime.

In Apex, a variable is a named memory location that stores a value of a specific data type. The data type of a variable determines the type of value it can hold, such as a string, number, or Boolean. Below are some examples of variables in Apex:


  • Integer variable: An integer variable stores whole numbers without decimal points. 

For example, Integer x = 10;.


  • String variable: A string variable stores a sequence of characters.

 For instance, String name = 'John';.


  • Boolean variable: A Boolean variable stores a value of either true or false.

 For example, Boolean flag = true;.


  • Double variable: A double variable stores decimal numbers with double precision.

 For example, Double pi = 3.14159;.


  • Date variable: A date variable stores a specific date. 

For instance, Date today = Date.today();.


  • Time variable: A time variable stores a specific time.

 For example, Time currentTime = System.now().time();.


  • Datetime variable: A datetime variable stores both a date and time. 

For instance, Datetime now = Datetime.now();.


  • Object variable: An object variable stores an instance of a specific class.

 For example, MyCustomObject__c obj = new MyCustomObject__c();.


  • List variable: A list variable stores a collection of values of a specific data type.

 For example, List names = new List{'John', 'Mary', 'Tom'};.


  • Map variable: A map variable stores a collection of key-value pairs of specific data types. 

For instance, Map grades = new Map{'John'=>90, 'Mary'=>80, 'Tom'=>85};.


In Apex, variables are declared with a specific data type and can be initialized with a value at the same time or later. They can be used to store and manipulate data throughout the program.

DML in Apex

DML (Data Manipulation Language) is a set of statements in Apex that are used to insert, update, delete, and retrieve records in the Salesforce database. Here are some examples of DML statements in Apex:

  1. Inserting Records:

Account acc = new Account(Name='Test Account', Industry='Technology'); insert acc;

This code creates a new Account record with a name of 'Test Account' and an industry of 'Technology', and inserts it into the database.

  1. Updating Records:

Account acc = [SELECT Id, Name, Industry FROM Account WHERE Name='Test Account' LIMIT 1]; acc.Industry = 'Healthcare'; update acc;

This code queries for an Account record with a name of 'Test Account', updates its industry to 'Healthcare', and updates the record in the database.

  1. Deleting Records:

Account acc = [SELECT Id, Name FROM Account WHERE Name='Test Account' LIMIT 1]; delete acc;

This code queries for an Account record with a name of 'Test Account' and deletes it from the database.

  1. Retrieving Records:

List accs = [SELECT Id, Name, Industry FROM Account WHERE Industry='Technology'];

This code queries for all Account records with an industry of 'Technology' and stores them in a list variable 'accs'.

DML statements can also be used in bulk to perform operations on multiple records at once. For example, the insert statement can be used with a list of records to insert them in bulk:

List accs = new List(); accs.add(new Account(Name='Test Account 1', Industry='Technology')); accs.add(new Account(Name='Test Account 2', Industry='Healthcare')); insert accs;

In summary, DML statements in Apex are used to manipulate records in the Salesforce database, and can be used to insert, update, delete, and retrieve records, both individually and in bulk.

Collection in Apex

In Apex, a collection is a data type that can store multiple values of the same data type. There are three types of collections in Apex: Lists, Sets, and Maps. Here are some examples of how to use each of these collection types:

  • Lists:

Lists are ordered collections of elements that can contain duplicates. Here's an example of how to create a List of Strings:

List myStrings = new List{'apple', 'banana', 'cherry'};

  • Sets:

Sets are collections of unique elements that are not ordered. Here's an example of how to create a Set of Integers:

Set myIntegers = new Set{1, 2, 3, 4};

  • Maps:

Maps are collections of key-value pairs where each key maps to a single value. Here's an example of how to create a Map of String keys and Integer values:

Map myMap = new Map{'apple' => 1, 'banana' => 2, 'cherry' => 3};

In addition to these basic examples, collections can be used in a wide variety of scenarios in Apex, such as storing query results, iterating through collections with for loops, and passing collections as parameters to methods. Using collections can help simplify code and make it more efficient.

In Apex, list, map, and set are collection types used to store and manage data in a structured way. Here are the definitions and examples of each:


  • List:

A list is an ordered collection of elements of the same type, such as integers or strings. Elements in a list are accessed by their index position. Lists can be modified by adding, removing, or updating elements.

lists in Apex include add(), remove(), size(), and get().

Example:


List names = new List{'Alice', 'Bob', 'Charlie'};

names.add('David');

names.remove('Bob');

names[1] = 'Eve';

In this example, we create a list of strings named 'names' and initialize it with three elements. We then add a new element 'David' to the list, remove the element 'Bob', and update the element at index 1 to 'Eve'.


  • Map:

A map is a collection of key-value pairs, where each key is unique and maps to a value. Maps can be used to store and retrieve data based on a specific key. Maps can also be modified by adding, removing, or updating key-value pairs.

Some of the commonly used methods for maps in Apex include put(), get(), remove(), and containsKey().

Example:


Map ages = new Map{'Alice'=>25, 'Bob'=>30, 'Charlie'=>35};

ages.put('David', 40);

ages.remove('Bob');

ages.put('Charlie', 36);


In this example, we create a map named 'ages' with string keys and integer values. We initialize it with three key-value pairs. We then add a new key-value pair 'David'=>40 to the map, remove the key-value pair with key 'Bob', and update the value of the key 'Charlie' to 36.


  • Set:

A set is an unordered collection of unique elements of the same type, such as integers or strings. Sets can be used to ensure that only unique elements are stored and to perform set operations such as union, intersection, and difference.

Some of the commonly used methods for sets in Apex include add(), remove(), size(), and contains().

Example:


Set colors = new Set{'red', 'green', 'blue'};

colors.add('yellow');

colors.remove('red');



In this example, we create a set of strings named 'colors' and initialize it with three elements. We then add a new element 'yellow' to the set and remove the element 'red'.

Insert triggers are fired when a new record is inserted into a Salesforce object. There are two types of insert triggers: before insert and after insert.

  • Before Insert: Before insert triggers are used to validate and modify the record data before it is saved to the database. In a before insert trigger, you can modify the record data, add or remove related records, or perform any other operation before the record is saved to the database.

  • After Insert: After insert triggers are used to perform actions after the record has been saved to the database. In an after insert trigger, you can perform any operation that requires the record to be saved to the database first.
