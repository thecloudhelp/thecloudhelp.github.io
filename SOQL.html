What is SOQL?

SOQL, or Salesforce Object Query Language, is a versatile querying language that enables users to retrieve and manipulate data from Salesforce databases. Specifically created to work with Salesforce's multi-tenant architecture, SOQL can efficiently query one or multiple objects.

Why SOQL?
SOQL enables you to retrieve specific data sets, apply filters based on criteria, and sort results from queries. Additionally, it lets you traverse objects and obtain related data from them. 

Although SOQL's syntax is similar to that of SQL, it contains special keywords and functions optimized for Salesforce's data model and object relationships.SOQL is a widely used tool for Salesforce developers and administrators to create custom reports, dashboards, and list views. It is also valuable for building custom integrations and automating business processes. 

Learning SOQL is crucial for enhancing your Salesforce skills and productivity, regardless of your proficiency level. By mastering SOQL, you can work with Salesforce data more effectively and efficiently.

New to Salesforce and SOQL?

How to open Developer Console in Salesforce?

 Composing your first query can feel overwhelming, but there's no need to fret. By following a few uncomplicated steps, you'll be able to write your inaugural SOQL query swiftly and confidently.

  • Step 1: Log in to your Salesforce account and open the Developer Console. This is the platform where you will write and execute your SOQL (Salesforce Object Query Language) query.
  • Step 2: Identify the object you want to query. For example, if you want to query accounts, the object will be "Account".
  • Step 3: Determine which fields you want to retrieve. For example, if you want to retrieve the account name and owner, you need to include the "Name" and "Owner" fields in your query.
  • Step 4: Compose your query. Your query will look like this: "SELECT Name, Owner FROM Account". This informs Salesforce to retrieve the account name and owner fields from the account object.
  • Step 5: Execute your query. Once you have written your query, click "Execute" to run it. The results of your query will appear in the "Query Results" pane at the bottom of the Developer Console.
  • Congratulations! You have just written your first SOQL query. As you gain more experience, you can write more complex queries and leverage Salesforce's data model and relationships between objects to retrieve and manipulate data in various ways.

Select statement in SOQL

SOQL's SELECT statement is a critical feature used to extract specific fields from one or multiple Salesforce objects. Despite sharing similarities with the SELECT statement in SQL, there are several important differences.

When utilizing the SELECT statement in SOQL, it is important to first specify the object that requires querying. For instance, if the aim is to extract data from the Account object, the query should begin with "SELECT," followed by the desired fields and the object from which they need to be fetched. Here is an example query that fetches the Account Name and Account Owner fields from the Account object:

SELECT Name, Owner FROM Account

To retrieve fields from related objects, you can use the SELECT statement. If you want to retrieve the Contact Name and Account Name for all contacts linked to an account, a parent-child subquery can be employed. The following query demonstrates how to fetch the Contact Name and Account Name fields for all contacts associated with the "ABC Company" account, utilizing a parent-child subquery:

SELECT Name, (SELECT Name, Account.Name FROM Contacts) FROM Account WHERE Name='ABC Company'

The above query employs the Account object as the parent object and the Contact object as the child object. The subquery extracts the Contact Name and Account Name fields from the Contacts object and the Account object, respectively.The SELECT statement is an effective tool that allows you to retrieve precise data from Salesforce objects. By combining the SELECT statement with other SOQL keywords and functions, you can retrieve, filter, and manipulate data to fulfill your unique business requirements.

Where Keyword in SOQL

SOQL (Salesforce Object Query Language) is a powerful tool that allows developers to retrieve data from Salesforce databases. One of the key features of SOQL is the ability to use the WHERE keyword to filter the data returned by a query.

The WHERE keyword in SOQL is used to specify one or more conditions that must be met in order for a record to be returned by the query. These conditions can be simple or complex and can involve multiple fields and operators.

For example, suppose you have a Salesforce database containing information about customers, including their names, addresses, and phone numbers. You could use the following SOQL query to retrieve all of the customers who live in New York:

SELECT Name, Address, Phone FROM Customer__c WHERE State = 'New York'

In this example, the WHERE keyword is used to specify the condition that the State field must be equal to 'New York' in order for a record to be returned. The query will only return records that meet this condition.The WHERE keyword can also be used in conjunction with other keywords, such as ORDER BY and LIMIT, to further refine the results of a query. For example, you could modify the previous query to return only the first 10 customers who live in New York and are sorted by their last name:

SELECT Name, Address, Phone FROM Customer__c WHERE State = 'New York' ORDER BY LastName LIMIT 10

In this modified query, the ORDER BY keyword is used to sort the results by the LastName field, and the LIMIT keyword is used to limit the number of results to 10.In conclusion, the WHERE keyword is a powerful tool in SOQL that allows developers to retrieve only the data they need from Salesforce databases. By specifying one or more conditions, developers can filter the results of a query to meet their specific needs. However, it is important to use the WHERE keyword correctly and with care to avoid unintended consequences and ensure the integrity of the data returned by the query.

Logical operator in SOQL

Logical operators are essential components in Salesforce Object Query Language (SOQL) that allow developers to retrieve specific data from Salesforce databases. There are three logical operators in SOQL: AND, OR, and NOT.

The AND operator in SOQL is used to retrieve records that satisfy multiple conditions. For example, if you want to retrieve all accounts that have an annual revenue of more than $1 million and are located in the United States, you can use the following SOQL query:

SELECT Id, Name, AnnualRevenue, BillingCountry FROM Account WHERE AnnualRevenue > 1000000 AND BillingCountry = 'United States'

NULL in SOQL

In SOQL, null values can be utilized in filters and can be checked with comparison operators such as =, !=, <, >, <=, and >=. The IS NULL operator is employed when filtering for null values. Consider the following SOQL query, which fetches all contacts without an email address.

SELECT Id, Name FROM Contact WHERE Email IS NULL

The query mentioned below fetches all contacts where the Email field has a null value. It is crucial to note that null values differ from empty or blank values.An empty or blank value signifies a value with no data or characters, while a null value indicates a value that does not exist. While querying data in SOQL, it is essential to distinguish between null values and empty or blank values, as it can significantly affect the results obtained from the query. In certain situations, null values can lead to SOQL errors. For instance, if a query attempts to access a null value in a field, it may result in an error. To prevent such errors, developers can employ the IFNULL() function to substitute null values with a specified default value. For example, the SOQL query mentioned below retrieves all accounts and substitutes null values in the Industry field with the default value "Not Specified".

SELECT Id, Name, IFNULL(Industry, 'Not Specified') FROM Account

In conclusion, null values in SOQL indicate the absence of a value in a field and can impact the results of a query. Developers must understand how to use null values in filters and comparison operators and differentiate between null values and empty or blank values to ensure the accuracy and integrity of their data. By using the IFNULL() function, developers can handle null values and avoid errors in their SOQL queries.

The LIKE operator in SOQL is employed to filter records based on a pattern that matches a specified string. It operates similarly to the SQL LIKE operator and enables developers to fetch records that contain a specific pattern in a particular field.

To use the LIKE operator in SOQL, the syntax is as follows:

SELECT field1, field2, ... FROM object WHERE field LIKE 'pattern'

This query will retrieve all contacts whose first name starts with the letter "J", such as "John", "Jane", and "James"

.In addition to the percent sign and underscore, the LIKE operator in SOQL supports the ESCAPE keyword. The ESCAPE keyword allows developers to search for strings that contain the percent sign or underscore characters as literal characters, rather than as wildcards.For example, the following SOQL query retrieves all accounts whose name contains the string "100%":


SELECT Id, Name FROM Account WHERE Name LIKE '%100!%%' ESCAPE '!'

In this query, the ESCAPE keyword is used with the exclamation mark (!) character to indicate that the percent sign (%) character should be treated as a literal character, rather than as a wildcard. The query will retrieve all accounts whose name contains the string "100%", such as "Acme Corporation 100% Satisfaction Guaranteed".In conclusion, the LIKE operator in SOQL is a powerful tool for filtering records based on a pattern that matches a specified string. By using the percent sign and underscore wildcards, as well as the ESCAPE keyword, developers can create complex queries that retrieve the precise data they need from Salesforce databases.

 LIMIT clause in SOQL

The LIMIT clause is used to limit the number of records returned by a SOQL query. Here's the syntax for using LIMIT in a SOQL query:

SELECT field1, field2, ... FROM object_name WHERE condition_expression LIMIT n

In the LIMIT clause, you specify the maximum number of records to return, where n is a positive integer.
 The LIMIT clause is optional, and if you don't specify a limit, Salesforce will return up to 2,000 records by default.

It's important to note that the LIMIT clause only limits the number of records returned by the query, not the number of records evaluated by the query.

 For example, if your query includes a filter that matches a large number of records, the query may still hit Salesforce's governor limits, even if you've specified a LIMIT clause.Here's an example of a SOQL query that includes a LIMIT clause

SELECT Id, Name, CreatedDate FROM Account WHERE CreatedDate > 2022-01-01T00:00:00Z LIMIT 10

OFFSET in SOQL

The OFFSET clause is an effective means of skipping a particular number of records before retrieving the remaining records in a query result. This feature is particularly useful when implementing pagination in an application or when retrieving only a subset of the query results.

Here's the syntax for using OFFSET in a SOQL query:

SELECT field1, field2, ... FROM object_name WHERE condition_expression ORDER BY field_name [ASC|DESC], ... LIMIT row_count OFFSET offset_count

In the following query, the OFFSET clause appears after the LIMIT clause. The row_count parameter specifies the maximum number of records to be retrieved, while the offset_count parameter indicates the number of records to be skipped before starting to retrieve records.To illustrate, suppose you want to retrieve the next 10 contacts in your Salesforce org after skipping the first 20 contacts. In this case, you can use the following SOQL query:

SELECT Id, FirstName, LastName FROM Contact ORDER BY LastName ASC LIMIT 10 OFFSET 20

GROUP BY clause in SOQL

The GROUP BY clause in a SOQL query is utilized to group the query results based on one or more fields. This is particularly useful when there's a need to aggregate data and perform calculations on subsets of records that share a common value in one or more fields.

The following is the syntax for using GROUP BY in a SOQL query:

SELECT field1,... FROM object_name WHERE condition_expression GROUP BY field_name, ...

When using the GROUP BY clause, you must specify the field(s) to be used for grouping, separated by commas.It is also possible to include aggregate functions such as COUNT(), SUM(), AVG(), MIN(), or MAX() to perform calculations on the grouped data. For example, suppose you want to obtain the total number of opportunities and the total amount for each stage in your Salesforce org. In that case, you can execute the following SOQL query:

SELECT StageName, COUNT(Id), SUM(Amount) FROM Opportunity GROUP BY StageName

The following query will group opportunities based on their stage name and provide the count of opportunities along with the sum of their amounts for each stage. It is crucial to note that when using GROUP BY, only fields specified in the GROUP BY clause or an aggregate function can be selected. Additionally, certain clauses such as HAVING, LIMIT, or OFFSET cannot be used in conjunction with GROUP BY.
